# telegram_bridge

텔레그램 메시지로 **Claude Code 작업을 원격 트리거**하는 브리지. 휴대폰에서
`<프로젝트명> <작업 지시>` 를 보내면 노트북의 `claude -p`(headless)가 해당 프로젝트에서
작업을 수행하고 결과를 회신한다. 확인 후 `push` 라고 답장하면 원격에 반영한다.

> PC 앞에 없어도 폰으로 "이 버그 고쳐줘" 한 줄이면 노트북이 대신 일하고, 커밋은 승인받고서만 올린다.

- **스택**: Python 3.13 **표준 라이브러리만**(외부 의존성 0) — Telegram Bot API 롱폴링 + `subprocess`
- **실행 호스트**: 상시 켜 둔 노트북 — 롱폴링이라 공개 서버·포트포워딩·호스팅 불필요
- **실시간 진행 표시**: 작업 중 메시지 하나가 `📖 읽음 · ✏️ 수정 · ⚡ 실행` 으로 라이브 갱신(stream-json)
- **탭 버튼**: 프로젝트 목록·`push` 확인을 인라인 키보드로 — 타이핑 없이 선택
- **실행비서 3기능**: 시각 알림 · 사진 대조 · 버튼 선택지(아래 참조)
- **품질**: 단위 테스트 181 · `ruff` · `mypy --strict` · 보안 감사(RCE 표면) 통과

## 동작 흐름

```
폰(텔레그램)  ──메시지──▶  브리지(노트북, 롱폴링)
                              │  ① chat ID 허용목록 인증
                              │  ② "<프로젝트> <지시>" 파싱 → 대상 폴더 해석
                              ▼
                          claude -p (headless, 해당 프로젝트에서 작업)
                              │  ③ 파일 수정 + 로컬 커밋
                              ▼
        결과 회신  ◀──────────┘
        "push" 답장 시 ──▶  git pull --rebase → push (원격 반영)
```

단일 롱폴링 루프가 메시지를 **직렬 처리**한다(작업 중 도착한 메시지는 텔레그램에
보존됐다 순차 처리 — 별도 큐·락 불필요).

작업 중에는 `🔄 작업 중…` 메시지 하나가 Claude의 도구 사용(읽기·수정·실행)을 따라
**실시간으로 갱신**되다, 끝나면 `[ ✅처리완료 ]` 결과로 바뀐다(2.5초 간격·최근 12줄).
프로젝트를 못 찾거나 형식이 안 맞으면 **대상 프로젝트를 탭 버튼**으로 띄우고,
로컬 커밋이 생기면 결과에 **[✅ Push] [❌ 취소]** 버튼을 붙여 타이핑 없이 반영·취소할 수 있다.

## 보안 설계

원격에서 코드를 실행·커밋하는 표면이라 방어를 다층으로 둔다.

| 경계 | 방어 |
|---|---|
| **접근 통제** | chat ID 허용목록 필수 — 목록 밖 메시지는 무회신·로그만(봇은 공개 검색될 수 있음). 빈 목록이면 기동 자체를 거부 |
| **명령 인젝션** | 사용자 메시지를 명령줄 인자에 **절대 두지 않고 stdin 으로만** 전달. Windows 에서 `claude` 는 배치 shim(`claude.CMD`)이라 인자가 `cmd.exe` 재파싱을 거치는데, stdin 전달로 그 표면을 원천 제거 |
| **권한 최소화** | claude 는 `--allowedTools` 화이트리스트(읽기·편집·특정 git·린트/테스트)만 — 일반 셸·`git push`·네트워크 미부여 |
| **비밀값** | 봇 토큰·chat ID 는 `.env` 로만(커밋 금지). 회신·로그에서 토큰·내부 경로는 마스킹 |
| **푸시 통제** | 작업은 로컬 커밋까지만. 원격 반영은 사용자가 `push` 라고 답장해 **명시 승인**했을 때만, `pull --rebase` 후 |
| **단일 인스턴스** | pidfile 락(같은 봇을 두 곳에서 폴링하면 텔레그램 409) |

## 설치 (3단계)

1. **봇 생성** — 텔레그램 [@BotFather](https://t.me/BotFather) 에 `/newbot` → 이름·유저네임
   지정 → 발급된 **봇 토큰** 복사.
2. **`.env` 작성** — `.env.example` 을 `.env` 로 복사해 채운다:
   - `TG_BOT_TOKEN` = 발급받은 토큰
   - `TG_ALLOWED_CHAT_IDS` = 본인 chat ID(콤마 구분). 봇에 아무 메시지나 보낸 뒤
     `https://api.telegram.org/bot<토큰>/getUpdates` 응답의 `message.chat.id` 로 확인
   - (선택) `CLAUDE_TIMEOUT_SEC`, `TARGET_ROOT` 는 기본값 그대로 두면 된다
3. **실행** — `run.bat`(또는 `python bridge.py`). 폰에서 `/help` 로 시작.

### 노트북 상시 운영

브리지는 노트북에서 도는 프로세스라, **켜 두는 동안만** 폰 원격이 동작한다.

- **절전 해제**(필수) — 전원 옵션에서 "절전: 안 함", "덮개를 닫을 때: 아무 동작 안 함".
  잠자면 롱폴링이 멈춘다.
- **자동 시작**(선택) — 부팅 시 자동 실행하려면 `run.bat` 바로가기를 시작프로그램 폴더
  (`Win+R` → `shell:startup`)에 넣는다.
- **한 번에 한 대만** — 여러 PC를 쓰면 브리지는 한 대에서만 실행한다(같은 봇을 두 곳에서
  폴링하면 텔레그램 409). pidfile 락이 같은 PC의 중복 실행은 막아 준다.

## 명령

| 입력 | 동작 |
|---|---|
| `<프로젝트명> <작업 지시>` | 해당 프로젝트에서 claude 작업 실행 후 결과 회신 (예: `trading_info README 오타 고쳐줘`) |
| `/projects` | 원격 지시 가능한 대상 프로젝트 목록(탭 버튼) |
| `push` | 로컬 커밋을 원격 main 에 반영(`pull --rebase` 후 push, 충돌 시 중단). 작업 후 **[✅ Push]** 버튼으로도 가능 |
| `/help` | 도움말 |

> 프로젝트명을 못 알아듣거나 형식이 안 맞으면 대상 프로젝트가 **탭 버튼**으로 표시된다
> (탭하면 그 프로젝트에 이어서 지시를 보내라는 안내). 버튼 탭도 chat ID 허용목록으로 통제된다.

## 실행비서 (능동 기능 3종)

원격 트리거를 넘어, 브리지가 **먼저 말을 걸고**·**보낸 사진을 읽고**·**버튼으로 응답을 받는** 세 기능.

### ① 시각 알림 — 스케줄 데이터 기반 능동 알림 + 버튼

`schedules/notify.json` 에 정의한 점검 항목을 정해진 요일·시각에 브리지가 **먼저 폰으로 알린다**.
각 알림에는 확인 버튼이 붙어, 탭 한 번으로 처리 여부를 기록한다(`grace_min` 안에 반응이 없으면 리마인드).

- 설정: `schedules/notify.example.json` 을 **`schedules/notify.json` 으로 복사**해 항목을 채운다.
- 항목 필드: `id` · `project` · `days`(요일) · `at`(HH:MM) · `label` · `note` · `grace_min`(분).
- `notify.json` 은 개인 점검 항목이라 커밋되지 않는다(예시 파일만 공개).

### ② 사진 대조 — 캡처를 보내면 로컬 값과 대조

폰에서 화면 캡처(예: 시세 앱)를 **사진으로 보내면**, 브리지가 로컬 REST(예: `trading_info` 의
`127.0.0.1:8000`)에서 해당 종목의 자체 값을 조회하고, **claude 의 이미지 판독으로 두 값을 대조**해
일치/불일치를 회신한다.

- 대조 프로세스의 claude 권한은 **Read 전용**(파일 수정·실행 없음) — 판독만 하는 최소 권한.
- 캡션의 티커만 사용하고, 라우팅은 REST 가 존재하는 프로젝트로 고정(다중화는 필요할 때 분기).

### ③ 버튼 선택지 — headless 세션을 이어 폰 버튼으로 응답

작업 중 claude 가 사용자에게 **선택지를 물으면**, 그 항목이 인라인 버튼으로 폰에 뜬다.
버튼을 탭하면 브리지가 **같은 세션을 `--resume` 으로 이어** 선택값을 전달해 작업을 계속한다
(타이핑 없이 진행). 버튼 응답도 chat ID 허용목록으로 통제된다.

## 개발

```bash
python -m pytest tests/     # 단위 테스트
ruff check . && ruff format --check .
python -m mypy --strict bridge.py
python bridge.py --selftest # 순수 함수 스모크(보안 경계 포함)
```

전 로직이 단일 파일 `bridge.py` 에 있다(롱폴링 루프 · 파싱 · 프로젝트 해석 ·
claude 러너 · git 오케스트레이션). 파싱·허용목록·경로 해석·마스킹 등 순수 함수는
분리돼 테스트로 고정돼 있다.
